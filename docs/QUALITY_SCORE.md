# QUALITY_SCORE

Методика оценки зрелости проекта «Бухта».

Статус: `Draft`.

## 1. Назначение

Документ нужен как краткая шкала для периодической оценки качества проекта, а не как backlog.

## 2. Шкала

- `0` — критично неработоспособно.
- `1` — фрагментарно, много ручных обходов.
- `2` — базово работает, но есть заметные риски.
- `3` — production-minimum для внутренней CRM.
- `4` — устойчиво и предсказуемо.
- `5` — хорошо автоматизировано и покрыто проверками.

## 3. Области оценки

- доменные инварианты;
- тестовое покрытие критичных сценариев;
- auth/roles;
- транзакции и история операций;
- frontend usability;
- dev/test/docs harness;
- Docker/local reproducibility.

## 4. Когда обновлять

После значимого изменения quality-contour: новый тестовый слой, первый production-like smoke, изменение auth, появление docs harness, крупная доменная операция.
