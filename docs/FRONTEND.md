# FRONTEND

Frontend notes проекта «Бухта».

Статус: `Draft`. Не фиксирует финальный UI до появления дизайна и первых экранов.

## 1. Назначение

Документ нужен для правил frontend-реализации после scaffold:

- структура web-приложения;
- server-state conventions;
- формы и валидация;
- UI primitives;
- тестирование пользовательских сценариев.

## 2. Уже принятые правила

- Frontend не является источником доменных инвариантов.
- Server-state держим в выделенном слое.
- Финальные проверки прав, остатков, денег, скидок и статусов делает backend.
- UI должен быть рабочим инструментом для внутренней команды, без маркетинговой подачи.

## 3. Что пока не фиксируем

- финальную навигацию;
- дизайн-систему;
- список экранов;
- визуальный стиль;
- component architecture сверх базовых принципов.

## 4. Заполнить после первых экранов

- layout conventions;
- form patterns;
- table/list patterns;
- error/loading/empty states;
- component test strategy.
