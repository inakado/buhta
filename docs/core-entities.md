# Основные объекты системы

Документ фиксирует базовые бизнес-объекты CRM и минимальный набор полей, нужный для понимания процесса. Это не схема базы данных и не техническое ТЗ.

## 1. Шаблон продукции

Шаблон продукции — заранее созданный тип продукции, по которому заведующий производством выпускает готовую продукцию.

Поля:

- название;
- цена за единицу;
- единица измерения / формат продажи;
- активен или скрыт.

Примеры:

- Икра горбуши 250 г;
- Икра кеты 500 г;

## 2. Сырье

Сырье — исходный ресурс, из которого выпускается готовая продукция.

Поля:

- вид сырья;
- количество;
- единица измерения;
- дата поступления;
- комментарий.

## 3. Выпуск продукции

Выпуск продукции — операция, в которой заведующий производством переводит сырье в готовую продукцию по выбранному шаблону.

Поля:

- выбранный шаблон продукции;
- сколько сырья использовано;
- сколько готовой продукции получилось;
- дата выпуска;
- ответственный заведующий производством;
- комментарий.

Результат операции:

- сырье уменьшается;
- готовая продукция на производстве увеличивается.

## 4. Готовая продукция

Готовая продукция — продукция, которая уже произведена и может быть передана логисту, на точку выдачи или продана через прямую отправку.

Поля:

- шаблон продукции;
- количество;
- цена за единицу из шаблона;
- текущее местоположение;
- статус.

Возможные местоположения:

- производство;
- у логиста;
- точка выдачи;
- прямая отправка.

## 5. Точка выдачи

Точка выдачи — объект системы и место учета, где хранится продукция и проводятся продажи клиентам. Точку выдачи создает администратор системы или директор. После создания на точку назначается работник точки выдачи.

Поля:

- название;
- тип точки;
- остаток продукции;
- денежный остаток;
- назначенный работник точки выдачи;
- ответственные сотрудники.

Типы точки выдачи:

- обычная точка выдачи.

Отдельно от точки выдачи существует канал «Прямая отправка»: через него коммерческий руководитель оформляет продажу, а логист доставляет продукцию сразу клиенту.

## 6. Остаток продукции точки выдачи

Остаток продукции точки выдачи показывает, какая продукция находится на точке выдачи и на какую сумму.

Поля по каждой позиции:

- шаблон продукции;
- количество;
- цена за единицу;
- сумма позиции.

При продаже:

- количество продукции уменьшается;
- сумма остатка продукции уменьшается;
- денежный остаток точки увеличивается.

## 7. Клиент

Клиент — покупатель, на которого оформляется продажа.

Поля:

- имя;
- телефон;
- описание / комментарий;
- кто добавил клиента.

Клиент обязателен для продажи.

## 8. Продажа

Продажа — операция передачи продукции клиенту.

Поля:

- клиент;
- точка выдачи или канал «Прямая отправка»;
- проданная продукция;
- количество;
- сумма продажи;
- кто оформил продажу;
- дата продажи.

Продажу может оформить:

- работник точки выдачи;
- коммерческий руководитель.

Логист продажу не оформляет.

## 9. Доставка / передача продукции

Доставка — задача для логиста переместить продукцию.

Поля:

- откуда забрать продукцию;
- куда отвезти продукцию;
- продукция и количество;
- связанная продажа, если это «Прямая отправка»;
- статус доставки;
- ответственный логист.

Логист видит задачи на доставку, готовую продукцию на производстве и остатки по точкам выдачи.

Открытый вопрос:

- кто формирует задачу на доставку: работник точки выдачи, логист или коммерческий руководитель?

## 10. Денежная передача

Денежная передача — движение денег от работника точки выдачи к логисту и далее к директору.

Поля:

- источник денег;
- получатель денег;
- сумма;
- дата передачи;
- статус;
- комментарий.

Цепочка денег:

Работник точки выдачи  
→ логист  
→ директор
